SIGNAL: marketplace-aware wrapper + admin hashlist download

Wrapper (signalInscription.ts)
- Replaces the previous minimal wrapper with the marketplace-aware
  template from signal-marketplace/wrapper-template.html.
- New per-mint footprint: 4184 bytes (up from 431).  Edition number is
  zero-padded to 4 digits so the byte length stays constant across all
  editions #0001..#1000 (verified for #1, #7, #42, #123, #999, #1000).
- Adds two new exported constants:
    SIGNAL_MARKETPLACE_INSCRIPTION_ID  — on-chain marketplace target
    SIGNAL_HASHLIST_SAT                — sat-pointer for hashlist updates

Hashlist service (services/signalHashlist.ts, NEW)
- buildSignalHashlist(adminAddress, onProgress) reconstructs the
  definitive SIGNAL hashlist purely from persistent sources:
    1. /api/admin/logs/all (techAndGames bucket, filtered for SIGNAL)
    2. /api/minting/resolved-ids (pending -> final id resolution)
    3. https://ordinals.com/r/inscription/<id> (definitive number + sat)
- Skips any still-pending mints, sorts confirmed ones by inscription
  number ascending, assigns gap-free editions 1..N.
- downloadSignalHashlist(hashlist) triggers a JSON download matching
  the schema the SIGNAL marketplace inscription expects.
- Build is idempotent: hashlist can never be lost because it is always
  derived from the persistent mint logs + chain data.

Admin Panel (components/AdminPanel.tsx, additive only)
- Adds a "📥 SIGNAL Hashlist" button in the Tech & Games minting-logs
  tab, next to the existing JSON / CSV download buttons.  Shows live
  progress while fetching ord recursive metadata, then downloads the
  rebuilt hashlist as signal-hashlist-vN-YYYY-MM-DD.json.
- No existing buttons, fields or layouts are removed or altered.

Wrapper byte stability and existing import surface
(SIGNAL_WRAPPER_HTML / SIGNAL_PREVIEW_SRCDOC / buildSignalWrapper /
SIGNAL_WRAPPER_BYTES) are preserved so collectionMinting.ts and
TechGamesPage.tsx keep working unchanged.
